# Implementation Plan - Account and Character Management Fix

Fix the issues with account and character management, enforce business rules (unique names, 50 chars limit), and improve the item addition UX with cascading selection.

## User Review Required

> [!IMPORTANT]
> **Data Consistency**: I will be standardizing the `my_items` table to use `account` and `character_name` as consistent references. If an account or character is renamed (not currently supported but good to keep in mind), the items will still reference the old names unless we migrate them. For now, I'll focus on the requested features.
> **Account Uniqueness**: Account names will be unique across the entire system (as per current schema `UNIQUE NOT NULL`). This matches the request "each account name must be unique".

## Proposed Changes

### Database & Initialization

#### [MODIFY] [app.py](file:///c:/OpenCode/D2RItemDB/app.py)
- Update `init_db()` to properly create the `characters` table:
  ```sql
  CREATE TABLE IF NOT EXISTS characters (
      id INTEGER PRIMARY KEY,
      account_id INTEGER NOT NULL,
      user_id INTEGER NOT NULL,
      name TEXT NOT NULL,
      class TEXT,
      created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      FOREIGN KEY (account_id) REFERENCES accounts (id),
      FOREIGN KEY (user_id) REFERENCES users (id),
      UNIQUE(account_id, name) -- Character name unique per account
  )
  ```
